# 本地视频压缩网页 — 设计文档

> 日期: 2026-07-11 | 状态: 待评审

## 1. 概述

一个纯浏览器端的视频压缩工具，单 HTML 文件（搭配本地 mp4box.js WASM），使用 WebCodecs API 调用 GPU 硬件编码器，支持分辨率/码率/帧率精确调控、预估压缩后大小、批量处理和压缩前后对比。

## 2. 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 视频解码 | WebCodecs VideoDecoder | 调用 GPU 硬解码 |
| 视频编码 | WebCodecs VideoEncoder | 调用 GPU 硬编码 (NVENC/AMF/QSV) |
| 音频编解码 | WebCodecs AudioDecoder / AudioEncoder | AAC 编码 |
| 容器解封装 | mp4box.js | 解析 MP4/WebM/MKV 容器 |
| 容器封装 | mp4box.js | 生成 MP4 容器文件 |
| 帧处理 | OffscreenCanvas / Canvas | 缩放、帧率控制 |
| UI | 原生 HTML/CSS/JS | 零框架依赖 |

目标浏览器: **Chrome 94+ / Edge 94+**（WebCodecs 完整支持）。

### 核心流程

1. 用户拖入视频文件 → mp4box.js 解析容器，提取视频/音频轨道信息
2. 读取元数据：原始分辨率、帧率、码率、编码格式、时长
3. VideoDecoder 解码为 VideoFrame → 送入 Canvas 做缩放/帧率调整
4. VideoEncoder 以用户指定的参数重新编码（分辨率/码率/帧率）
5. AudioDecoder → AudioEncoder 处理音频
6. mp4box.js 将编码后的视频+音频流封装为 .mp4 文件
7. 触发浏览器下载

## 4. 模块设计

### 4.1 文件管理模块

- **拖拽区**: 页面顶部大面积拖拽区域，支持 `dragenter` / `dragover` / `drop` 事件，附带文件选择按钮
- **队列**: 任务列表，每项显示文件名、原始信息、目标参数、预估大小、状态/进度
- **支持容器**: `.mp4` `.webm` `.mkv` `.mov` `.avi`
- **支持编码**: H.264 / H.265 / VP8 / VP9 / AV1（不支持的编码标记为"无法解码"）
- **批量导入**: 一次拖入多个文件全部加入队列

### 4.2 参数面板

每个文件独立参数，选中文件后右侧面板显示；提供"应用到全部"按钮。

| 参数 | 控件 | 说明 |
|------|------|------|
| 目标分辨率 | 预设按钮 + 自定义输入 | 4K(3840×2160) / 1080p / 720p / 480p / 原尺寸 / 自定义宽×高 |
| 视频编码器 | 下拉选择 | H.264 (普遍可用) / H.265 (需硬件支持) / VP9 / AV1，不支持的灰掉 |
| 码率模式 | 单选 | VBR（质量优先）/ CBR（精确大小控制） |
| 码率值 | 范围滑块 + 数字输入 | 500 Kbps ~ 50 Mbps，根据分辨率给出推荐默认值 |
| 帧率 | 下拉选择 | 保持原帧率 / 60 / 30 / 24 / 15 fps |
| 音频比特率 | 下拉选择 | 128 / 192 / 256 Kbps，AAC 编码 |

### 4.3 预估大小

**计算公式:**
```
预估大小(MB) = (视频码率(Kbps) + 音频码率(Kbps)) × 时长(秒) / 8 / 1024 × 1.02
```

- 1.02 为 MP4 容器开销系数（约 2%）
- UI 显示范围: `预估: 45.3 MB (约 42.8 ~ 47.6 MB)`，±5%
- 参数变化时实时更新

### 4.4 压缩引擎 (CompressionEngine)

核心类，封装完整编码管线：

```
class CompressionEngine {
  constructor(file, options, onProgress, onComplete, onError)
  async start()
  abort()
}
```

**管线阶段:**
1. `demux()` — mp4box.js 解析容器，获取 EncodedVideoChunk / EncodedAudioChunk
2. `decode()` — VideoDecoder / AudioDecoder 解码为原始帧/样本
3. `process()` — Canvas 缩放 + 帧率控制（丢帧/补帧）
4. `encode()` — VideoEncoder / AudioEncoder 重新编码
5. `mux()` — mp4box.js 封装为 MP4 文件

**进度回调**: `onProgress({stage, percent, fps, estimatedRemaining})`

### 4.5 预览与对比

- 原始预览: `<video>` 元素直接播放选中文件
- 压缩后对比 (完成后出现):
  - 两个 `<video>` 并排显示
  - 显示统计: 原始大小 / 压缩后大小 / 节省百分比 / 压缩比
  - 滑块对比模式: 覆盖播放，拖动滑块分界线

## 5. 错误处理

| 场景 | 处理 |
|------|------|
| 浏览器不支持 WebCodecs | 页面顶部红色横幅，提示使用 Chrome 94+ 或 Edge 94+ |
| 编码器硬件不支持 (如 H.265) | 下拉选项灰掉，hover 提示 "当前设备不支持 H.265 硬件编码" |
| 文件不是有效视频 | Toast "文件 xxx 不是有效的视频格式" |
| 编码中途失败 | 任务标记失败，显示错误信息，不影响队列其他文件 |
| 内存不足警告 | 编码前粗略估算所需内存，超阈值弹警告确认 |
| 页面关闭确认 | `beforeunload` 事件，有进行中任务时提示用户 |

## 6. 并发控制

- 默认同时编码 **1 个文件**（GPU 编码器实例有限，并行可能反而慢）
- 高级设置中可调整为 N 个并行（适合 CPU 软编码降级场景）

## 7. 兼容性检测

页面加载时自动检测：
1. `VideoEncoder` / `VideoDecoder` / `AudioEncoder` / `AudioDecoder` 是否存在
2. 各编码器配置是否支持 (`isConfigSupported`)
3. 动态构建可用编码器列表

## 8. 依赖来源

- **mp4box.js**: 从 [github.com/gpac/mp4box.js](https://github.com/gpac/mp4box.js) 获取 `dist/mp4box.all.js` 和 `dist/mp4box.wasm`
- 两个文件放置在与 `index.html` 同目录，HTML 通过 `<script>` 标签加载
- 版本锁定为 v0.5.2（稳定版，已验证 WebCodecs 配合使用）

## 9. 容器格式支持说明

mp4box.js 以 ISOBMFF (MP4) 为核心支持：

| 容器格式 | 解封装 | 封装 | 说明 |
|----------|--------|------|------|
| .mp4 | ✅ | ✅ | 完全支持 |
| .webm | ✅ (有限) | ❌ | 部分 WebM 文件可解封装; 所有输出统一为 MP4 |
| .mkv | ✅ (有限) | ❌ | Matroska 可通过 mp4box.js 实验性支持读取 |
| .mov | ✅ (有限) | ❌ | MOV 与 MP4 结构相似，多数能解析 |
| .avi | ❌ | ❌ | AVI 容器不支持，提示用户先用其他工具转换 |

用户选择非 MP4 输入时，页面顶部显示提示 "非 MP4 文件的兼容性取决于源容器结构"。

## 11. 非功能需求

- 所有资源从本地加载，无需网络连接
- 视频数据全程在浏览器内处理，不上传任何数据
- 响应式布局，适配 1024px 及以上宽度
- 编码进度实时展示，速度显示 FPS 或实时倍数
